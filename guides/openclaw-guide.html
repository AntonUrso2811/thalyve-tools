1. What OpenClaw Actually Is

OpenClaw is the bridge between you and your AI agent. It's a gateway that:

  • Connects to your chat channels (Telegram, WhatsApp, Discord, etc.)
  • Routes your messages to an AI model (Claude, GPT, etc.)
  • Gives the AI tools (file access, web browsing, code execution, etc.)
  • Runs on your machine ‚Äî your data stays yours

In plain English: It's the app that makes your AI agent live inside your Telegram (or wherever). Without it running, no messages get through.

4. API Key Setup ‚Äî Anthropic (Primary)

Getting Your API Key

  1. Go to console.anthropic.com
  2. Sign up / log in
  3. Go to API Keys ‚Üí Create new key
  4. Copy the key (starts with sk-ant-...)
  5. Add billing (pay-as-you-go, no subscription needed)

Installing the Key

# Option 1: Via the wizard (easiest)
openclaw onboard

# Option 2: Direct command
openclaw onboard --anthropic-api-key "sk-ant-YOUR-KEY-HERE"

# Option 3: Manual config edit
openclaw config set env.ANTHROPIC_API_KEY "sk-ant-YOUR-KEY-HERE"

Setting the Model

# Set Claude Opus as your primary model
openclaw config set agents.defaults.model.primary "anthropic/claude-opus-4-6"

Verify It Works

openclaw models list
openclaw gateway restart
Cost Info
Anthropic API is pay-per-use. Claude Opus costs roughly $15/million input tokens, $75/million output tokens. Typical daily usage for one person: $2-10/day depending on how much you chat. There's no monthly subscription ‚Äî you only pay for what you use.

7. Gateway ‚Äî Starting, Stopping, Debugging

The gateway is the heart of OpenClaw. If it's not running, nothing works.

Essential Commands

# Check if gateway is running
openclaw gateway status

# Start the gateway
openclaw gateway start

# Stop the gateway
openclaw gateway stop

# Restart (use after any config change)
openclaw gateway restart

# Force start (kills anything on the port first)
openclaw gateway --force

# See live logs
openclaw logs --follow

# Full health check
openclaw doctor

The Debug Ladder

When something isn't working, run these in order:

# 1. Is it running?
openclaw gateway status

# 2. Overall health?
openclaw status

# 3. What do the logs say?
openclaw logs --follow

# 4. Auto-diagnose
openclaw doctor

# 5. Are channels connected?
openclaw channels status --probe
Golden Rule
After any config change (API key, model, channel), always run: openclaw gateway restart

8. Common Issues & Fixes

Issue: "Gateway not running"

openclaw gateway start
# If it won't start:
openclaw gateway --force
# Still stuck? Check what's on the port:
lsof -i :18789

Issue: "Port already in use" / EADDRINUSE

# Force-kill anything on the port
openclaw gateway --force
# Or manually:
kill $(lsof -t -i :18789) 2>/dev/null
openclaw gateway start

Issue: "Unauthorized" / Token Mismatch

# Check your gateway token
openclaw config get gateway.auth.token
# Reset if needed
openclaw configure

Issue: "No replies from bot"

# Check if your Telegram ID is paired/approved
openclaw pairing list telegram
# Check routing
openclaw status
openclaw logs --follow
# Look for: "pairing request" or "blocked" or "allowlist"

Issue: API Key Not Working

# Verify key is set
openclaw config get env.ANTHROPIC_API_KEY
# Re-set it
openclaw config set env.ANTHROPIC_API_KEY "sk-ant-YOUR-KEY"
openclaw gateway restart

Issue: "Path not found" / Misaligned Paths

# Check where OpenClaw thinks home is
echo $HOME
ls -la ~/.openclaw/

# Check config file location
openclaw config get --path

# If paths are wrong, reset config location
export OPENCLAW_STATE_DIR="$HOME/.openclaw"
openclaw gateway restart

Issue: Model Not Responding / Errors

# Check which model is set
openclaw config get agents.defaults.model.primary

# List available models
openclaw models list

# Switch to backup while debugging
openclaw config set agents.defaults.model.primary "openai/gpt-4o"
openclaw gateway restart

Issue: "gateway.mode=local" Error

# Set local mode
openclaw config set gateway.mode "local"
openclaw gateway restart

10. Security & Proxies ‚Äî Do You Need Them?

Short Answer
No, you do not need proxies. You are using a legitimate, paid Anthropic API key. There is zero risk of being "shut down" for normal use. Proxies would add latency and complexity for no benefit.

What IS Worth Protecting

RiskProtectionStatus
API key exposedStored securely in OpenClaw config (not in code/repos)‚úÖ Done
Unauthorized access to botTelegram pairing ‚Äî only approved users can chat‚úÖ Done
Gateway exposed to internetDefault: loopback only (localhost)‚úÖ Default
Machine compromisedFirewall, SSH hardening, updates‚è≥ Audit pending

Security Audit (When Ready)

openclaw security audit

This checks config, permissions, exposed services, and gives recommendations.

When You WOULD Need Proxies

  • If you were using a shared or leaked API key (you're not)
  • If the API was geo-restricted and you needed to route from another country (it's not)
  • If you were scraping or doing something against ToS (you're not)
Bottom Line
Pay for your API key ‚Üí use it normally ‚Üí don't worry about it. Anthropic wants you to use their API. That's how they make money.

11. Keeping It Running 24/7

Current Setup (Mac)

OpenClaw installs itself as a background service (launchd on Mac). It auto-starts on boot.

# Check service status
openclaw gateway status

# If it stopped, restart
openclaw gateway restart

Option A: Old MacBook as Always-On Server

  • Pros: Free, your data stays local, no monthly cost
  • Cons: Depends on your internet/power, needs to stay open and connected
  • Setup: Plug it in, disable sleep, start gateway, done
# Disable sleep on Mac (run on the server Mac)
sudo pmset -a disablesleep 1
sudo pmset -a sleep 0

# Verify
pmset -g | grep sleep

Option B: VPS (Recommended Long-Term)

  • Cost: ~¬£4-5/month (Hetzner, Contabo, or similar)
  • Pros: Always on, professional-grade uptime, accessible from anywhere
  • Cons: Small monthly cost, data on someone else's server
# On a fresh VPS (Ubuntu):
curl -fsSL https://openclaw.ai/install.sh | bash
openclaw onboard
openclaw gateway start
Recommendation
Start with the Mac (it's working now). Move to VPS when you want bulletproof reliability. I'll handle the entire migration when you're ready.

13. Emergency Recovery

If everything is broken and you need to start fresh:

Soft Reset (Keeps Your Data)

# Stop everything
openclaw gateway stop

# Run the doctor
openclaw doctor

# Reconfigure
openclaw configure

# Restart
openclaw gateway start

Hard Reset (Nuclear ‚Äî Last Resort)

# ‚ö†Ô∏è This resets ALL config (not workspace files)
openclaw reset

# Then set everything up again
openclaw onboard
openclaw gateway start
Warning
openclaw reset wipes your config (API keys, channel settings, etc.) but keeps your workspace files (SOPs, memory, etc.) safe. You'll need to re-enter your API key and bot token after.

Full Reinstall (Absolute Last Resort)

# Uninstall
openclaw uninstall

# Reinstall
curl -fsSL https://openclaw.ai/install.sh | bash
openclaw onboard
Your Data Is Safe
Your workspace (~/.openclaw/workspace/) survives resets and reinstalls. All your SOPs, memory files, and documents will still be there. Only config needs to be re-entered.

14. Subscription Auth (Claude Max/Pro ‚Äî Flat Rate)

Instead of paying per-token (API key), you can use a Claude Max subscription ($200/month flat) for unlimited usage. Two methods:

Method A: Setup Token (Native ‚Äî Recommended)

Uses Anthropic's official subscription auth directly in OpenClaw.

1Install Claude Code CLI 
npm install -g @anthropic-ai/claude-code
2Log in to your Claude subscription 
claude login
3Generate a setup token 
claude setup-token

This outputs a token. Copy it.

4Paste it into OpenClaw 
# Option 1: Interactive
openclaw onboard --auth-choice setup-token

# Option 2: Direct paste
openclaw models auth paste-token --provider anthropic

# Then restart
openclaw gateway restart
Setup Token Notes
  • Tokens can expire ‚Äî if you get 401 errors, re-run claude setup-token and paste again
  • Prompt caching does NOT work with subscription auth (only API keys)
  • If you see "OAuth token refresh failed", re-auth with a fresh setup-token

Method B: Claude Max API Proxy (Community Tool)

Runs a local proxy that turns your Claude subscription into an OpenAI-compatible API. Useful if setup-token has issues.

1Install the proxy 
npm install -g claude-max-api-proxy
2Start it 
claude-max-api
# Runs at http://localhost:3456
3Point OpenClaw at it 
openclaw config set env.OPENAI_API_KEY "not-needed"
openclaw config set env.OPENAI_BASE_URL "http://localhost:3456/v1"
openclaw config set agents.defaults.model.primary "openai/claude-opus-4"
openclaw gateway restart
4Auto-start on boot (Mac) 
# Create a launch agent so it runs automatically
cat > ~/Library/LaunchAgents/com.claude-max-api.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key><string>com.claude-max-api</string>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><true/>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/node</string>
    <string>/usr/local/lib/node_modules/claude-max-api-proxy/dist/server/standalone.js</string>
  </array>
</dict>
</plist>
EOF

launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.claude-max-api.plist

15. Terms of Service & Compliance Protocol

Are You Breaking Any Rules?

Short Answer
No. Everything you're doing is fully within terms of service. Paid API key, personal bot, private machine. This is exactly how these services are designed to be used.

What's Allowed ‚úÖ

  • Using paid API keys for personal/business automation
  • Running AI agents on your own machine
  • Connecting to Telegram/WhatsApp/Discord bots
  • Using the AI to write copy, build SOPs, manage files
  • Running multiple models from different providers
  • Using subscription auth (setup-token) for personal projects
  • Having the AI access your local files, web, tools

What to Avoid ‚ùå

  • Sharing API keys ‚Äî never give your key to anyone or post it publicly
  • Reselling API access ‚Äî don't let others use your agent as a service
  • Scraping/abuse ‚Äî don't use the API to mass-scrape websites or spam
  • Harmful content ‚Äî don't use it to generate illegal, deceptive, or harmful content
  • Impersonation ‚Äî don't use AI-generated content to impersonate real people
  • Circumventing rate limits ‚Äî don't use multiple accounts to bypass usage limits

If Something Goes Wrong

API key compromised?

# 1. Revoke immediately on console.anthropic.com
# 2. Generate new key
# 3. Update OpenClaw
openclaw config set env.ANTHROPIC_API_KEY "sk-ant-NEW-KEY"
openclaw gateway restart

Unexpected charges?

  1. Check Anthropic console ‚Üí Usage page
  2. Revoke key if unauthorized use detected
  3. Set billing limits/alerts on the console
  4. Rotate to new key

Account flagged or suspended?

  1. Contact Anthropic support directly
  2. Switch to backup model (OpenAI/Gemini) while resolving
  3. Review what triggered it (usually automated ‚Äî false positives happen)
Action Item
Right now: Go to console.anthropic.com ‚Üí Settings ‚Üí Billing and set a monthly spending limit/alert. This is your safety net. Even $50/month alert gives you early warning.

Privacy & Data Handling

  • Anthropic API: Your data is NOT used for training (API terms). Conversations are transient.
  • OpenClaw: Runs locally on your machine. No data sent to OpenClaw servers (open source).
  • Telegram: Messages route through Telegram's servers (encrypted in transit). Bot messages are stored by Telegram.
  • Your workspace files: Stay on your Mac. Not synced anywhere unless you choose to.
Bottom Line
You're operating 100% within bounds. The biggest real risk is accidentally leaking your API key ‚Äî and that's already protected. Everything else is clean.
